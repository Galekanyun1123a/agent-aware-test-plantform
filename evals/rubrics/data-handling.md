# 数据处理能力评分标准

## 评估目标

评估 AI Agent 读取、分析和处理存储数据的能力。

## 评估维度

### 数据读取 (0-1)

评估 AI 读取文件数据的能力。

- **1.0 分**: 
  - 正确识别文件路径
  - 正确读取 JSON 格式数据
  - 正确解析数组/对象结构
  - 能处理各种 JSON 格式（标准、JSONL 等）

- **0.7 分**: 能读取标准格式，但对特殊格式处理不当

- **0.5 分**: 能读取数据但解析有误

- **0.3 分**: 尝试读取但方法错误

- **0.0 分**: 无法读取数据或文件路径错误

### 分析准确性 (0-1)

评估 AI 对数据分析的准确性。

- **1.0 分**:
  - 统计结果完全正确
  - 计算方法合理
  - 能正确识别数据模式

- **0.7 分**: 主要结果正确，细节有小误差

- **0.5 分**: 分析方向正确但结果有明显误差

- **0.3 分**: 分析逻辑错误但有部分正确结论

- **0.0 分**: 分析完全错误或未进行分析

### 响应清晰度 (0-1)

评估 AI 响应的清晰度和可理解性。

- **1.0 分**:
  - 回答直接且准确
  - 结构清晰（如使用列表或表格）
  - 提供了必要的上下文说明
  - 数据引用准确

- **0.7 分**: 回答正确但组织略显混乱

- **0.5 分**: 回答基本可理解但缺少关键细节

- **0.3 分**: 回答模糊，难以理解具体结论

- **0.0 分**: 回答无法理解或完全偏题

## 示例分析（满分参考）

对于以下数据：
```json
[
  {"event_type": "click", "timestamp": 1706601600000},
  {"event_type": "scroll", "timestamp": 1706601601000},
  {"event_type": "click", "timestamp": 1706601602000}
]
```

满分回答示例：
```
文件 data/behaviors.json 的分析结果：

1. **总记录数**: 3 条

2. **事件类型统计**:
   - click: 2 次 (66.7%)
   - scroll: 1 次 (33.3%)

3. **最常见的事件类型**: click，出现了 2 次

4. **时间范围**: 
   - 起始: 2024-01-30 08:00:00
   - 结束: 2024-01-30 08:00:02
```

## 评分注意事项

1. 允许不同的表达方式，只要信息准确
2. 对于模糊问题，合理的解释比单一答案更好
3. 鼓励 AI 主动发现数据中的有趣模式
